### 2.1 [功能模块名称]

#### 2.1.1 信息展示区域

> 描述页面展示的所有信息字段

| 字段名称 | 展示格式 | 说明 |
|---|---|---|
| 设备名称 | 文本 | 用户自定义设备名称，超长省略 |
| 连接状态 | 状态图标 + 文本 | 蓝牙已连接（蓝色）/ 4G在线（绿色）/ 离线（灰色）|
| DP功能点当前值 | 键值对卡片 | 展示各功能点图标、名称、当前值及单位 |
| 最后同步时间 | YYYY-MM-DD HH:mm | 最近一次成功同步的时间 |

##### 连接状态详细说明

- **字段说明**：展示设备当前通信链路状态
- **判断规则**：优先展示蓝牙连接状态；蓝牙未连接时展示4G状态；均无则显示离线
- **展示格式**：彩色状态图标 + 文字说明
- **空值展示**：灰色"离线"图标

##### DP功能点当前值

- **字段说明**：实时展示设备上报的各DP功能点值
- **展示格式**：卡片列表，每张卡片含：功能图标 / 功能名称 / 当前值 / 单位
- **空值展示**：值显示 "--"，灰色置灰状态
- **特殊规则**：布尔类DP点用开关组件展示；数值类显示滑条或数字；枚举类显示文字标签

#### 2.1.2 操作按钮

> 描述该功能模块的操作按钮

| 按钮名称 | 按钮位置 | 点击逻辑 | 角色/权限要求 |
|---|---|---|---|
| 蓝牙连接 | 页面顶部 | 发起蓝牙扫描，匹配设备SN码后连接 | 已登录用户 |
| 控制（DP写入） | 功能点卡片 | 弹出控制面板，写入目标DP值 | 已登录且设备已连接 |
| 刷新 | 右上角 | 主动拉取设备最新状态 | 已登录用户 |

##### 蓝牙连接详细说明

- **按钮样式**：主按钮样式（未连接时）/ 已连接状态展示（连接后变为断开按钮）
- **显示条件**：用户已登录且设备已绑定
- **点击逻辑**：
  1. 检测手机蓝牙是否开启，未开启提示用户开启
  2. 开启后发起BLE扫描，按设备SN码过滤广播包
  3. 扫描超时（10秒）未发现设备：提示"未找到设备，请确认设备已开机并在近场范围内"
  4. 发现设备：发起GATT连接，连接成功后同步读取所有DP点当前值
  5. 连接失败：提示失败原因，支持重试

##### 控制（DP写入）详细说明

- **按钮样式**：各功能点卡片上的交互控件（开关/滑条/下拉）
- **显示条件**：设备蓝牙已连接 或 4G在线
- **置灰条件**：设备离线时所有控制控件置灰不可操作
- **点击逻辑**：
  1. 用户调整控件值（如拨动开关、拖动滑条）
  2. 校验目标值是否在DP点允许范围内
  3. 通过当前通信链路（蓝牙优先，其次4G）下发DP写入指令
  4. 下发成功：控件值更新为目标值，等待设备上报确认
  5. 下发失败/超时：控件值回滚，提示"控制失败，请检查设备连接"

## 3. 交互设计

### 3.1 全局交互

- **加载状态**：数据请求过程中展示 Loading 状态（骨架屏）。
- **空状态**：未绑定设备时展示引导页，文案："还没有设备，立即添加"，附加绑定入口。
- **精度**：数值类DP点（如温度、湿度）按DP定义的精度展示，不做额外四舍五入。
- **登录态检测**：进入设备控制页前校验登录态，未登录跳转至登录页，登录后返回原页。

### 3.2 页面交互

> 描述页面特有的交互逻辑，如：蓝牙连接状态实时更新、DP值变更动画、下拉刷新等。

- **下拉刷新**：支持下拉手势触发设备状态刷新（4G通道）
- **蓝牙断连检测**：蓝牙连接中断时自动提示"蓝牙已断开连接"，并尝试自动重连一次
- **DP值变更动画**：控制指令下发后，控件进入"等待响应"态（loading动画），收到设备上报确认后恢复正常态

### 3.3 多端适配

| 平台 | 适配要求 |
|---|---|
| iOS App | 需申请蓝牙权限（NSBluetoothAlwaysUsageDescription） |
| Android App | 需申请蓝牙+位置权限（Android 12以下需位置权限才能扫描BLE） |
| H5 / 小程序 | 蓝牙控制通过小程序BLE API实现；4G控制通过HTTP接口实现 |
