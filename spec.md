# PRD Generator — 配套规格文档

本文档定义 `prd-generator` Skill 的技术规格、模板结构约定与输出标准，供维护者参考。

---

## 1. 模板类型规格

### 1.1 类型枚举

| 类型标识 | 中文名 | 模板文件 | 适用场景 |
|---|---|---|---|
| `data` | 数据看板类 | `assets/templates/data-prd-template.md` | 图表、统计卡片、Dashboard、运营数据看板 |
| `backend-list` | 后台列表类 | `assets/templates/backend-list-prd-template.md` | 数据列表、搜索筛选、批量操作；含表单编辑页（附录E）和详情预览页（附录P）|
| `user-frontend` | 用户前台类 | `assets/templates/user-frontend-prd-template.md` | C端设备控制、骑行轨迹、智能服务等面向终端用户的页面 |

### 1.2 类型识别优先级

当页面特征存在交叉时，按以下优先级判断：

1. 含大量图表/指标卡 → `data`
2. 含数据列表/搜索筛选/分页 → `backend-list`
3. 面向C端用户 → `user-frontend`
4. 无法判断 → 询问用户

---

## 2. 输入文件规格

| 文件 | 必须 | 说明 |
|---|---|---|
| `index.tsx` / `index.vue` | 推荐 | 页面主组件，提取功能点和交互逻辑 |
| `spec.md` | 推荐 | 接口约定、DP功能点定义、数据结构 |
| `content.md` | 可选 | 文案、字段说明、业务规则 |
| `screenshot.png` / 截图 | 可选 | 布局和UI元素参考 |

> 至少需要以上文件之一，否则无法生成有效PRD。

---

## 3. 输出文件规格

| 项目 | 规范 |
|---|---|
| 格式 | Markdown（`.md`）|
| 文件名 | `prd.md`（小写）|
| 保存路径 | `src/prototypes/<page-name>/prd.md` |
| 语言 | 与用户沟通语言一致，默认中文 |
| 编码 | UTF-8 |

## 4. PRD 章节结构规格

### 4.1 通用必填章节（三种模板均包含）

| 章节 | 说明 |
|---|---|
| `1.1 背景` | 不超过50字的核心定位描述 |
| `1.2 功能清单` | 模块/功能点/描述三列表格，不含顶栏和菜单栏样式 |
| `2.x 权限规划` | 仅定义权限点和操作范围，不指定具体角色 |
| 需求说明主体 | 按页面类型对应章节填写 |
| 交互设计 | 全局交互（加载/空状态/精度）+ 页面特有交互 |

### 4.2 权限点命名约定

| 操作类型 | 权限点名称 |
|---|---|
| 查看、查询、导出详情 | 【查看】|
| 新增、编辑、删除、下发指令 | 【编辑】|
| 导出列表/报表 | 【导出】|

## 5. DP 功能点规格（物联网专项）

### 5.1 DP 点类型映射

| DP类型 | 前端展示控件 | 说明 |
|---|---|---|
| Boolean | 开关（Switch）| 开/关两态 |
| Integer / Float | 数字输入框 / 滑条（Slider）| 需标注单位和取值范围 |
| Enum | 下拉选择（Select）/ 分段控制 | 需列出所有枚举值及含义 |
| String | 输入框（Input）| 需标注最大长度 |
| RAW | 不可直接展示，需协议解析后展示 | |

### 5.2 通信链路优先级

```
蓝牙（BLE）近场  >  4G 远程
```

- 蓝牙已连接时，所有 DP 写入指令优先走蓝牙通道
- 蓝牙未连接时，走 4G 通道（需设备在线）
- 两者均不可用时，控制控件置灰，不可操作

## 6. 一致性核对检查项

生成PRD后必须自检以下三个维度，全部通过方可交付：

### 维度1：功能点与页面实现一致
- PRD中描述的功能在页面中都存在
- 没有描述页面中不存在的功能
- 交互逻辑与实现一致
- 展示规则与实现一致

### 维度2：技术约定与 spec.md 一致
- 数据口径定义一致
- 字段格式定义一致
- 接口规则定义一致
- 无与 spec.md 冲突的内容

### 维度3：范围与用户要求一致
- 功能覆盖了用户指定的范围
- 没有超出范围的内容

